데이터베이스 : 애플리케이션이 입력/처리/출력해야 되는 데이터 집합체
DBMS(데이터베이스 관리 시스템) : 애플리케이션이 입력/처리/출력해야 되는 데이터를 저장하고 관리,운영하는 시스템
관계형 데이터베이스 : 데이터 저장이 테이블과 같은 표로 이루어진 데이터베이스 
테이블 : 행과 컬럼으로 구성된 데이터 저장 구조
행(row) : 한 줄 데이터에 해당되며 레코드(Record),튜플(Tuple),엔티티(Entity) 라고도 한다
	각기 다른 여러 컬럼이 모인 한 줄 데이터
컬럼(coumn) : 한 칸 데이터에 해당되며  필드,항목(Item),속성(Attribute) 라고도 한다 
	     관계형 데이터베이스의 최소 단위 
	     
기본키(기본 항목) : Primary Key -레코드와 레코드를 식별할 수 있도록 설정한 필드
유일성, 최소성,Not Null -> 개체 무결성(레코드와 레코드를 식별함에 있어 중복되거나 널 값인 경우 식별이 불가능해지지 않도록 하는 특성)

외래키(참조 항목) : Foregin Key - 참조하려고 하는 테이블에 존재하는 비교(참조) 필드
반드시 참조 테이블에는 해당 값(참조대상)이 모두 있어야 함
참조 무결성 : 참조하려는 하는 테이블의 해당하는 값이 모두 존재해야 한다.
ex) 사원코드(각 코드가 달라), 직위코드(같은 직위는 코드가 같아)			    

뷰(veiw) : 실제 물리적으로 존재하는 것이 아닌 계산식이나 처리문장에 의하여 생성된 가상 테이블
